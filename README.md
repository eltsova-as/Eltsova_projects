# AB-тестирование нового алгоритма рекомендаций в развлекательном приложении с «бесконечной» лентой
### Описание проекта
**Контекст:** Развлекательное приложение с функцией «бесконечной» ленты (аналог коротких видео) имеет две модели монетизации: платная подписка (без рекламы) и рекламная модель для пользователей без подписки. Команда разработки создала новый алгоритм рекомендаций, который предположительно показывает более интересный контент для каждого пользователя.

**Цель исследования:** Рассчитать параметры А/В-теста, оценить корректность его проведения и проанализировать результаты эксперимента для принятия решения о внедрении нового алгоритма. 

**Основные задачи:**
* Провести анализ исторических данных (EDA)
* Сформулировать гипотезы и определить ключевую метрику
* Рассчитать необходимый размер выборки и длительность теста
* Провести мониторинг первых дней эксперимента (проверка корректности распределения)
* Проанализировать результаты теста и оценить статистическую значимость
* Сформулировать рекомендации по внедрению

**Ключевая метрика:** Доля успешных сессий (просмотр 4+ страниц за сессию) – прокси-метрика удовлетворенности контентом и вовлеченности

**Стек:** Python (pandas, matplotlib, SciPy, StatsModels), Jupyter Notebook, A/B-тестирование, статистический анализ (z-тест, t-тест, Манна-Уитни)

**Topics:** Python, pandas, matplotlib, SciPy, StatsModels, A/B-тестирование, проверка гипотез, продуктовая аналитика, z-тест, размер выборки, MDE, power анализ, визуализация данных

### Итоговые выводы

| Параметр | Значение |
|:---------|---------:|
| Общее количество пользователей | 99 405 |
| Контрольная группа (А) | 49 551 |
| Тестовая группа (В) | 50 454 |
| Длительность эксперимента | 9 дней |
| MDE (Minimum Detectable Effect) | 3% |
| Статистическая мощность | 80% |

**Результаты воздействия:**
* Статистически значимый положительный эффект (р=0.0001257)
* Абсолютный прирост ключевой метрики: +1.06%
* Относительный прирост: +3.43%

**Качество проведения теста:**
* Равномерное распределение пользователей по группам (разница <1%)
* Отсутствие пересечений (группы независимы)
* Сбалансированность по устройствам и регионам
* Сопоставимость групп по количеству сессий (p>0.05)

**Рекомендации по внедрению:**
Рекомендуется внедрить новый алгоритм рекомендаций на всю пользовательскую базу.
Обоснование:
* Статистическая значимость – p-value значительно ниже порога 0.05
* Практическая значимость – прирост ключевой метрики +3.43% имеет бизнес-ценность
* Качество эксперимента – тест проведен корректно, результаты валидны
* Влияние на бизнес – рост вовлеченности повышает вероятность регистрации и монетизации

# Superset: Аналитический дашборд маркетплейса по продажам
### Описание проекта

**Контекст:** Крупная онлайн-платформа для продаж, где малые и средние продавцы размещают свои товары. В компании десятки товарных категорий, тысячи продавцов, активные отделы продаж и маркетинга, которые ежедневно принимают решения на основе данных.

**Цель проекта:** Разработать комплексный аналитический дашборд в Apache Superset для отслеживания ключевых показателей эффективности маркетплейса, оптимизации ассортимента и удержания пользователей.

**Основные задачи:**
* Провести сбор требований и задокументировать ТЗ
* Выполнить предобработку данных в Python (pandas)
* Провести ad-hoc аналитику по запросам бизнеса
* Создать два ключевых датасета для дашборда
* Разработать макет в Miro и реализовать дашборд в Superset
* Провести тестирование и подготовить финальную документацию
* Обучить пользователей работе с дашбордом

**Стек:** Apache Superset, PostgreSQL, Python (pandas, matplotlib, seaborn), Jupyter Notebook, SQL, Miro

**Topics:** Superset, дашборды, маркетплейс, e-commerce, Python, pandas, SQL, визуализация данных, продуктовая аналитика, обучение пользователей

### Заключение

Разработанный дашборд предоставляем команде маркетплейса полный набор инструментов для:
* Мониторинга ключевых показателей (GMV, заказы, пользователи, рейтинги)
* Анализа эффективности продаж по категориям, товарам и регионам
* Оценки пользовательской активности (Retention, регистрации, поведение)
* Контроля качества через отзывы и рейтинги
* Принятия data-driven решений по ассортименту и маркетингу

Дашборд прошел тестирование, снабжен документацией для пользователей.

# Superset: Дашборд - Анализ рынка компьютерных игр
### Описание проекта

**Контекст:** Компания, специализирующаяся на разработке игр, нуждается в комплексном аналитическом инструменте для отслеживания трендов на рынке интерактивных развлечений. Данные охватывают период с 2007 по 2025 год. Дашборд предназначен для двух групп пользователей:
* Топ-менеджеры – для оценки потенциала новых релизов и корректировки ценовой политики
* Аналитики – для анализа эффективности отдельных игр и выявления точек роста

**Цель проекта:** Разработать интерактивный дашборд в Apache Superset, который позволит отслеживать ключевые метрики игрового рынка:
* Динамику цен на игры
* Пользовательский интерфейс
* Вовлеченность игроков в прохождение игр

**Стек:** Apache Superset, SQL, Jinja-шаблоны, виртуальные датасеты, CSS-стилизация

**Topics:** Superset, дашборды, визуализация данных, игровая аналитика, Jinja, YoY, MoM, когорты, Heatmap, геймдев, UI/UX, CSS

### Заключение

Разработанный дашборд предоставляет команде игровой компании полный набор инструментов для:
* Мониторинга ценовой динамики на рынке игр
* Анализа популярности игр через установки и достижения
* Оценки вовлеченности игроков (количество и редкость достижений)
* Сегментации аудитории по странам, платформам и жанрам
* Когортного анализа выпуска игр на платформах
* Отслеживания динамики YoY и MoM с цветовой индикацией

Благодаря использованию Jinja-шаблонов дашборд оптимально работает с большим объёмом данных, а гибкая система фильтров и продуманный UI/UX позволяют анализировать информацию в любом необходимом разрезе.
